# Девятая домашняя работа

## Были добавлены аутентификация и авторизация с помощью JWT
### Pinia
Создан стор auth, отвечающих за хранение и манипуляции с JWT.
### Navigation Guards
Допуск к странице проверяется через мета-данные путей. В компонентах проверяется только необходимость отрисовки тегов по роли/входу (например кнопка создать новый продукт для админов или копка создать новый пост для всех аутентифицированных).
### Проверка жизни JWT
Там же в Navigation Guards проверяется поле exp. Пользователя разлогинят и переведут на страницу входа как только дата превысит exp.
### Страница входа пользователей
Есть поля для логина и пароля, доступна только не аутентифицированному пользователю. Аутентифицированного пользователя страница входа переводит на главную страницу, если не было целевой. Со страницы идёт запрос к стору user на проверку логина и пароля. При удачной аутентификации, user делает запрос к стору auth на запись JWT. При неудачной аутентификации выводится ошибка "неверный логин или пароль".
### Профиль пользователя
Доступна только аутентифицированному пользователю. Отображает логин пользователя (из стора user), аватарку, имя и email с gql-сервера. Содержит кнопку выхода, которая посылает запрос в стор user на выход пользователя. Стор посылает запрос в auth стор на удаление jwt. Пользователя перекидывает на страницу входа.
### Страница, доступная только админу
Создавать новые продукты по адресу /newProduct может только администратор.
### Редирект на целевую страницу
Страница логина после удачной аутентификации переводит на целевую страницу, если таковая есть.
